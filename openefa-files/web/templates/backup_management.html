Backup Information
About Backups
  • Full System Backups: Complete backup including configuration files, database, and Python modules (tar.gz archive)
  • Database Backups: Full backup of the spacy_email_db database including all tables, routines, triggers, and events
  • Config Backups: Configuration files created automatically before modifications
  • Compression: All backups are automatically compressed to save space
  • Location: All backups are stored in /opt/spacyserver/backups/
Best Practices
  • Create backups before making major configuration changes
  • Keep at least 7 days of daily backups
  • Download critical backups to a secure off-site location
  • Test restore procedures periodically
  • Monitor backup sizes to track database growth
Note: Restore operations should be performed by system administrators via the OpenSpacyMenu or command line. Contact support if you need to restore from a backup.
